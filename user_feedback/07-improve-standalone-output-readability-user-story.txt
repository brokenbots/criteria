User Story: Improve standalone output readability and progress visibility
Date: 2026-04-26

As an operator running overseer in standalone mode,
I want a human-readable output mode that shows steps and current progress without full request/message payloads,
so that I can quickly understand run status without parsing verbose event data.

Current pain:
- Standalone output is very verbose and difficult to scan.
- Full request and message/event payloads overwhelm the important signal.
- It is hard to quickly see which step is active, what completed, and what failed.

Acceptance criteria:
- A concise output option (for example a flag) is available in standalone mode.
- Concise mode shows current step, step transitions, outcomes, and overall run progress.
- Verbose payloads (full request/message data) are hidden by default in concise mode.
- Operators can still opt into full event payloads when deep debugging is needed.
