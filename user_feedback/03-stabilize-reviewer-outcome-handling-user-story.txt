User Story: Stabilize reviewer outcome handling for Copilot runs
Date: 2026-04-26

As a workflow author,
I want resilient outcome mapping for common Copilot result variants,
so that minor wording differences do not hard-fail a run.

Current pain:
- Reviewer emitted needs_review, but the workflow had no mapped transition for that outcome.
- The run failed with unmapped outcome, even though the intent was clearly "continue iteration".

Acceptance criteria:
- Outcome aliases (for example needs_review -> changes_requested) can be configured or normalized.
- Unmapped outcome errors include nearest known outcomes and suggested transitions.
- Optional strict mode keeps current behavior for teams that want hard enforcement.
