User Story: Make for_each safe for multi-step iteration chains
Date: 2026-04-26

As a workflow author,
I want for_each to preserve each bindings across multi-step per-item flows,
so that realistic executor/reviewer/cleanup loops can run within a single iteration.

Current pain:
- Current engine behavior effectively requires the do step to return _continue.
- Transitioning from do to another step causes early exit semantics and clears each bindings.
- Later steps in the chain cannot access each.value and fail at runtime.

Acceptance criteria:
- each.value and each.index remain available throughout the iteration subgraph.
- The loop advances only when the iteration subgraph reaches a declared continue edge.
- Runtime/compile validation explains unsupported loop shapes before execution.
