User Story: Support file() in workflow expressions
Date: 2026-04-26

As a workflow author,
I want to load local file contents directly from workflow expressions (for example with file()),
so that I can reuse agent profiles, prompts, and templates without shell preload steps.

Current pain:
- file() is not available in the expression evaluator.
- We had to add shell pre-steps (cat/awk) to load and sanitize .agent.md files.
- This adds complexity and fragile parsing logic to otherwise declarative workflows.

Acceptance criteria:
- A supported expression function can read files from workspace-relative paths.
- Missing files produce a clear compile-time or runtime error with path context.
- Optional helpers exist for frontmatter stripping and newline normalization.
